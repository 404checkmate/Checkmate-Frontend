# CHECKMATE 프론트엔드 — 비개발자를 위한 프로젝트 이야기

---

## 이 서비스가 뭔가요?

**CHECKMATE(체크메이트)** 는 여행 준비를 도와주는 웹 서비스입니다.

여행지, 동행인, 여행 스타일을 입력하면 AI가 필요한 준비물 체크리스트를 자동으로 만들어줍니다.
만들어진 체크리스트는 저장하고, 수정하고, 하나씩 체크하면서 사용할 수 있습니다.

---

## 어떤 기술을 사용했나요?

### 집에 비유하면 이렇습니다

> 집을 짓는다고 생각해보세요.
> - **뼈대** — 구조를 잡아주는 기둥과 벽
> - **인테리어** — 보이는 디자인과 색상
> - **전기/수도** — 보이지 않지만 동작하게 만드는 시스템

우리 프로젝트도 마찬가지입니다.

---

### 뼈대: React + Vite

**React**는 화면을 만드는 도구입니다.

웹 화면을 "컴포넌트"라는 작은 조각들로 나눠서 만듭니다.
예를 들어 헤더, 버튼, 카드, 모달창 — 각각이 하나의 컴포넌트입니다.
레고 블록처럼 조각을 만들어두고 필요한 곳에 끼워 넣는 방식입니다.

**Vite**는 이 조각들을 빠르게 조립해서 브라우저에서 볼 수 있게 해주는 도구입니다.
개발 중에 코드를 수정하면 브라우저가 거의 즉시 반영됩니다.

---

### 인테리어: Tailwind CSS

**Tailwind CSS**는 디자인을 입히는 도구입니다.

일반적으로 디자인을 적용하려면 별도의 스타일 파일을 따로 만들어야 합니다.
Tailwind는 그 과정 없이 HTML 코드에 바로 디자인 클래스 이름을 붙여서 적용합니다.

예를 들어:
- `bg-teal-600` → 배경색을 청록색으로
- `rounded-2xl` → 모서리를 둥글게
- `text-white` → 글자색을 흰색으로

이 클래스 이름들을 조합해서 디자인을 완성합니다.

---

### 페이지 이동: React Router

**React Router**는 페이지 간 이동을 담당합니다.

웹사이트에서 메뉴를 클릭하면 다른 페이지로 이동하는 기능입니다.
주소창의 주소가 바뀌면서 그에 맞는 화면을 보여줍니다.

예를 들어:
- `/` → 홈페이지
- `/trips/new` → 새 여행 만들기 페이지
- `/checklist` → 체크리스트 페이지

---

### 드래그 앤 드롭: dnd-kit

**dnd-kit**은 마우스로 항목을 잡아서 끌고 다니는 기능을 담당합니다.

체크리스트에서 준비물 순서를 바꿀 때 사용됩니다.
항목을 클릭하고 위아래로 드래그하면 순서가 바뀌는 그 기능입니다.

---

### 서버 통신: Axios

**Axios**는 서버와 데이터를 주고받는 도구입니다.

우리가 로그인하거나 체크리스트를 저장할 때, 화면에서 일어나는 일은 사실 서버(백엔드)와 끊임없이 대화하는 것입니다.
Axios는 그 대화를 담당하는 "메신저" 역할입니다.

---

### 인증(로그인): Supabase

**Supabase**는 로그인 기능을 담당합니다.

카카오, 구글 등 소셜 로그인을 연결하고 사용자 세션을 관리합니다.
직접 로그인 시스템을 처음부터 만들면 매우 복잡하기 때문에, Supabase라는 전문 서비스를 연결해서 사용했습니다.

---

### 날짜 선택: react-day-picker + date-fns

**react-day-picker**는 달력 UI를 담당합니다.
여행 날짜를 선택할 때 나오는 달력이 이 도구로 만들어졌습니다.

**date-fns**는 날짜 계산을 담당합니다.
"출발일로부터 몇 일 남았는지", "여행 기간이 며칠인지" 같은 계산을 쉽게 해주는 도구입니다.

---

## 홈페이지 디자인, 어떻게 만들었나요?

홈페이지에는 크게 두 가지 시각적 효과가 있습니다.

---

### 효과 1. 섹션이 한 장씩 넘어가는 효과 (스크롤 스냅)

홈페이지에서 스크롤을 내리면 화면이 자연스럽게 다음 섹션으로 딱 맞춰 멈춥니다.
마치 파워포인트 슬라이드를 넘기는 것처럼요.

**어떻게 구현했나요?**

이 효과는 외부 라이브러리 없이 **브라우저 자체 기능**만으로 구현했습니다.

1. CSS에 `scroll-snap-type: y mandatory` 한 줄을 추가합니다.
   이것은 "세로 방향으로 스냅하되, 반드시 스냅 포인트에서 멈춰라"는 명령입니다.

2. 각 섹션에 `snap-start` 클래스를 붙입니다.
   이것은 "이 섹션의 시작점이 멈춰야 할 지점이다"라고 지정하는 것입니다.

3. 이 효과는 **홈페이지에서만** 적용됩니다.
   다른 페이지로 이동하면 자동으로 꺼집니다.
   React가 홈페이지를 열 때 `<html>` 태그에 클래스를 붙이고,
   홈페이지를 떠날 때 그 클래스를 제거하는 방식입니다.

4. **마지막 섹션(캐치프레이즈 + 푸터)** 은 두 개를 하나로 묶어서
   그 사이에서는 스냅 없이 자연스럽게 이어지도록 처리했습니다.

> 접근성 고려: 애니메이션에 민감한 사용자(모션 감소 설정을 켠 경우)는
> 스냅 효과가 자동으로 꺼지고 일반 스크롤로 동작합니다.

---

### 효과 2. 섹션이 뷰포트에 들어올 때 등장하는 효과 (스크롤 등장 애니메이션)

각 섹션이 화면에 들어오는 순간, 요소들이 아래에서 위로 부드럽게 나타납니다.
그리고 한 번 나타난 요소는 다시 스크롤해도 반복되지 않습니다.

**어떻게 구현했나요?**

두 가지 도구를 조합했습니다.

**1. IntersectionObserver (브라우저 내장 기능)**

"이 요소가 화면에 보이기 시작했나?"를 감지하는 브라우저 기능입니다.
감시카메라처럼 요소를 지켜보다가, 화면에 일정 비율 이상 들어오면 신호를 보냅니다.

```
요소가 화면에 18% 이상 들어오면 → "보임" 신호 발생 → 한 번만 실행
```

**2. CSS Transition (브라우저 내장 기능)**

"보임" 신호가 오기 전: 투명하고 40px 아래에 위치
"보임" 신호가 온 후: 불투명하고 제자리에 위치

이 두 상태 사이를 1.18초 동안 부드럽게 전환합니다.

**3. RevealBlock 컴포넌트**

위 두 가지를 매번 작성하기 번거로워서, `RevealBlock`이라는 컴포넌트 하나로 묶었습니다.

```
<RevealBlock show={보임여부}>
  여기 안에 있는 내용이 등장 애니메이션을 갖게 됩니다
</RevealBlock>
```

**순차 등장 효과**도 만들었습니다.
카드 3장이 동시에 나타나는 게 아니라 560ms 간격으로 하나씩 나타납니다.
각 RevealBlock에 `delay-[420ms]`, `delay-[980ms]`, `delay-[1540ms]` 클래스를 붙이는 것으로 구현합니다.

### 홈페이지 배경 디자인

배경은 단순한 단색이 아니라 여러 원형 빛이 겹쳐진 그라데이션입니다.

```
연한 민트 계열 배경
  + 오른쪽 위 노란 빛 (amber)
  + 왼쪽 아래 하늘색 빛 (cyan)
  + 가운데에서 아래로 색이 변하는 그라데이션
```

이것은 CSS의 `radial-gradient`(원형 그라데이션)와 `linear-gradient`(선형 그라데이션)를 겹쳐서 만든 것입니다.

## 홈페이지 섹션 구성

홈페이지는 총 5개 섹션으로 구성됩니다. 스크롤할 때마다 한 섹션씩 보입니다.

| 순서 | 섹션 이름 | 내용 |
|------|-----------|------|
| 1 | Hero (히어로) | 메인 타이틀 + 마스코트 이미지 + 시작하기 버튼 |
| 2 | Feature (기능 소개) | AI가 질문하는 말풍선 UI + 기능 설명 |
| 3 | Process (사용 방법) | 저장 → 확인 → 체크 3단계 카드 |
| 4 | CTA (행동 유도) | 마무리 문구 + 마스코트 일러스트 + 시작 버튼 |
| 5 | Catchphrase + Footer | 캐치프레이즈 + 하단 정보 (하나로 묶여 함께 스크롤) |

## 개발 방식 — 어떻게 팀이 작업했나요?

### 컴포넌트 분리

각 화면을 작은 조각(컴포넌트)으로 나눠서 작업했습니다.

예를 들어 홈페이지 하나도 이렇게 분리되어 있습니다:

```
HomePage.jsx (전체 조립)
  ├── HomeHeroSection.jsx       (첫 번째 섹션)
  ├── HomeFeatureSection.jsx    (두 번째 섹션)
  ├── HomeProcessSection.jsx    (세 번째 섹션)
  ├── HomeCtaSection.jsx        (네 번째 섹션)
  ├── HomeCatchphraseSection.jsx (다섯 번째 섹션)
  ├── HomeFooter.jsx            (푸터)
  └── HomeScrollToTopFab.jsx    (위로 가기 버튼)
```

이렇게 나누면:
- 각자 맡은 부분만 수정할 수 있습니다
- 한 파일이 너무 길어지지 않습니다
- 나중에 수정할 때 어디를 고쳐야 하는지 바로 찾을 수 있습니다

### 빌드 검증

코드를 수정하고 나면 항상 `npx vite build` 명령으로 최종 빌드가 정상인지 확인했습니다.
이 명령은 실제 배포할 파일을 만드는 과정으로, 오류가 있으면 여기서 잡힙니다.

| Axios | 서버 통신 | 백엔드와 데이터 주고받기 |
| Supabase | 인증 | 소셜 로그인 및 세션 관리 |
| react-day-picker | 달력 | 여행 날짜 선택 UI |
